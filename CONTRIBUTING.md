# Contributing to BabelLayer

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to **BabelLayer**. These are just guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
Bugs are tracked as GitHub issues. When creating a bug report, please include:
- A clear and descriptive title.
- Steps to reproduce the bug.
- A description of the behavior you observed.
- The environment you're running (OS, Python version, etc.).

### Suggesting Enhancements
Enhancements are also tracked as GitHub issues. When suggesting an enhancement:
- Use a clear and descriptive title.
- Provide a step-by-step description of the suggested enhancement.
- Explain why this enhancement would be useful.

### Pull Requests
- Fill in the required template.
- Ensure all tests pass.
- Follow the existing code style (clean, readable, and well-commented).

## Style Guide

### Python
We aim for clean, human-readable Python code.
- Follow PEP 8 where appropriate.
- Use descriptive variable and function names.
- Keep functions small and focused on a single task.

### Documentation
- Ensure new features are documented in the `docs/` folder.
- Update the `README.md` if necessary.

## Feedback
We appreciate all feedback! Feel free to open a discussion or contact the maintainers directly.
